## Context

In 1982 a researcher named David Eddy gave 100 physicians a reasoning problem about mammogram screening. The test caught 89% of cancers and produced false positives in 7% of healthy women, in a population where 0.35% actually had cancer. Eddy asked: a woman gets a positive mammogram, what's the probability she has cancer? The doctors averaged around 75%. The correct answer is about 4%.

The math the doctors didn't have is the math you'll learn this week. It's how a forensic DNA match becomes evidence at a trial, how a positive COVID test gets interpreted, how the FDA reads a drug-screening study. The framework, called Bayes' Theorem, lets you turn a positive test into a probability that the test is right. Used carefully it's how you tell a true positive from a false alarm. Used carelessly it sends innocent people to prison and panics healthy patients.

OpenIntro Section 3.2 walks through the exact same mammogram problem the doctors got wrong (Example 3.42 on page 105). This week you'll learn the textbook's approach, then apply it yourself.

## Steps

### Step 1: Read the assigned chapter, in this order

Open OpenIntro Statistics 4e, Chapter 3, Section 3.2 (pages 95 to 109). The reading is short, about 14 pages. Read it in this order; the order matters:

1. **Section 3.2.1 (page 95)**, "Exploring probabilities with a contingency table." A contingency table is a two-way summary that organizes counts by two variables. This is the central tool of the unit.
2. **Section 3.2.7 (page 102)**, "Tree diagrams." Tree diagrams are the second representation; they're better than contingency tables when probabilities come in sequence (first this, then that).
3. **Example 3.42 (page 105)**, the mammogram example. Walk through the entire worked solution. This is the problem the doctors got wrong, solved correctly. The exact same logic applies to Problem 1 in this week's set.
4. **The "Bayes' Theorem" callout box (page 106)**. This is the formal name for what you just did with the tree diagram. Don't try to memorize the formula. Notice instead that it does the same thing the tree diagram does.

Sections 3.2.2 through 3.2.6 are useful background but not strictly required for this week's problems. Read them if you have time.
